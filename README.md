# campus-canteen-fullstack
校园食堂小程序外卖点餐平台

## 📖 项目简介
本项目是一套**前后端分离**的校园食堂线上点餐全栈系统，分为「学生微信小程序点餐端」与「食堂商家Web管理后台」，适配高校食堂运营场景，解决线下排队、订单管理繁琐、经营数据无统计等痛点。
学生可在线浏览菜品、加购购物车、微信支付下单；食堂管理员可完成菜品管理、订单处理、实时接单播报、营收报表导出、用户权限管控等全套运营操作。

## ✨ 核心功能模块
### 学生微信小程序端
- 首页菜品分类、热销菜品推荐、食堂公告展示
- 购物车增删、菜品规格选择、自提地址管理
- 微信支付V3下单、订单状态实时查看、退款申请
- 个人中心：历史订单、个人信息、消费记录查询

### 食堂管理后台（Vue+ElementUI）
- 员工账号、角色分级权限管理（JWT双端鉴权）
- 菜品分类维护、菜品上下架、菜品图片上传阿里云OSS
- 实时订单看板：WebSocket来单弹窗+语音播报提醒
- 订单全流程处理：接单、出餐、完成、退款审核
- 数据可视化大盘：日/月营收、菜品销量统计图表
- Apache POI 营收、订单明细Excel报表导出
- 系统参数配置、缓存清理、操作日志查看

## 🎯 核心技术亮点
1. **双端JWT无状态鉴权体系**
区分食堂管理员、小程序学生两套Token签发逻辑，自定义拦截器配合ThreadLocal存储登录上下文，自动拦截未登录、越权访问接口，保障系统访问安全。

2. **Redis + SpringCache 缓存高并发优化**
整合声明式缓存对菜品列表、分类、热销榜单等高频查询接口加速，设计合理过期淘汰策略，实现缓存穿透防护，有效降低MySQL查询压力。

3. **WebSocket 实时订单推送播报**
前后端长连接通信，新订单实时推送至管理后台，自动弹窗+前端语音播报，无需轮询刷新页面，大幅提升食堂接单处理效率。

4. **微信支付V3 完整业务闭环**
完整实现预下单、支付异步回调、订单退款、超时未支付自动关单逻辑；校验回调请求签名，实现订单状态幂等更新，避免重复扣款、订单数据错乱。

5. **AOP切面+自定义注解简化CRUD**
通过切面自动填充创建人、创建时间、更新人、更新时间公共字段，消除业务层重复赋值代码，减少冗余开发代码。

6. **Redis分布式锁 + 乐观锁防并发问题**
下单流程使用分布式锁控制并发，结合数据库乐观锁管控订单状态流转，避免多人同时下单造成库存错乱、重复下单问题。

7. **Nginx 前后端分离线上部署**
配置静态资源长期缓存、后端接口反向代理，支持多服务实例负载均衡，分离前后端资源，提升页面访问速度。

## 🚀 本地快速启动
### 环境前置准备
- JDK 1.8+
- MySQL 8.0
- Redis 5.0+
- Node.js 14+（运行Vue后台）
- 微信开发者工具（调试小程序）

### 1. 数据库初始化
1. 新建数据库 `campus_canteen`
2. 执行项目 `/sql/canteen.sql` 文件，导入表结构与测试数据

### 2. 后端服务启动
1. 修改 `application.yml` 中 MySQL、Redis、阿里云OSS、微信支付配置信息
2. 运行启动类 `CanteenServerApplication`
3. 访问在线接口文档：`http://localhost:8080/doc.html`

### 3. Vue管理后台启动
```bash
# 进入后台目录
cd canteen-admin-vue
# 安装依赖
npm install
# 本地运行
npm run dev
